2) Current Capabilities

Market Data

fetch_ticker and fetch_candles return pandas DataFrames (1m/5m/15m).

Candle Builder

Tick-to-OHLC for 1-minute and 5-minute candles.

Strategies

Momentum and RSI scalping with SL/TP/trailing metadata.

Risk

1% risk per trade, 10x max leverage, 5% max daily loss validation.

Execution

Market/limit order execution, retry logic, stop/tp/trailing triggers.

Persistence

state.db lifecycle logging with unique execution IDs.

Backtesting

PnL, win rate, max drawdown, profit factor.

Monitoring

Prometheus: trade_count, win_rate, drawdown, api_latency.

3) Architecture

[Delta API/WS] | v [api/delta_client.py] -> [data/market_data.py] -> [strategy/*] | | | v | [risk/risk_management.py] | | v v [execution/order_execution_engine.py] | v [persistence/db.py] | +--> [monitoring/prometheus_exporter.py] Main runner: [cli/trading_bot.py] Legacy runner: [core/engine.py + cli/main.py]

4) Runtime Flow (One-Minute Loop)

  1. Fetch candles for symbol via fetch_candles(..., "1m").
  2. Compute EMA20 and RSI.
  3. Process open-position protection triggers (SL/TP/trailing).
  4. If no open position, generate strategy signal.
  5. Apply risk sizing and validation.
  6. Execute order (paper or live).
  7. Persist execution entries and open-position state.
  8. Record metrics and continue next cycle.

14) Troubleshooting

        
Auth issues: verify DELTA_API_KEY, DELTA_API_SECRET, and DELTA_EXCHANGE_ENV.

        
No signal: ensure enough candle history is available for EMA/RSI warm-up.

        
No trailing exits: ensure on_price_update is called regularly and protection state is active.
